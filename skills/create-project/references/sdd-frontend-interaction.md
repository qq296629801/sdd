# IIDP 前端交互设计规格模板

## 定位与使用时机

本文是"交互行为规格"模板，专注于**用户流程、交互状态、响应式行为、可访问性验收**，不输出节点 id、selector、ds_config、bind_ 等技术细节。

| 场景 | 用哪个模板 |
|---|---|
| 从业务需求/用户故事生成前端规格 | 用 `skills/frontend/references/iidp-frontend-spec-doc/SKILL.md` 的模板 |
| 需要输出节点树、selector、数据源、绑定、命令等技术细节 | 用 `sdd-frontend.md` |
| 需要页面级交互状态、响应式、可访问性验收 | 用**本文**模板 |

**分工边界**：

- **本文**：交互行为（"用户怎么用、状态怎么变、响应式怎么配、可访问性怎么验"）
- `sdd-frontend.md`：技术落地（"节点怎么写、绑定怎么配、数据源怎么连"）

两个模板配合使用，不重复输出。先用本文描述交互行为，再用 `sdd-frontend.md` 输出技术规格。

## 交互规格核心要素

| 要素 | 说明 | 输出位置 |
|---|---|---|
| 用户流程 | 进入页面 → 操作 → 成功/失败分支 | §2 |
| 页面目标 | 目标用户、核心任务、推荐实现方式 | §1 |
| 交互状态 | 各区域/组件的状态及触发条件 | §3 |
| 响应式行为 | 容器、滚动、弹窗尺寸策略 | §4 |
| 可访问性 | 表单标签、错误提示、键盘操作、危险操作确认 | §5 |
| 验收标准 | 可操作、可验证的检查点 | §6 |

**本文不输出**：节点 id、selector、ds_config、bind_、bind_on_、commands、hook 路径——这些在 `sdd-frontend.md` 输出。

## 规格文档模板

以下为完整的交互设计规格文档模板。生成规格时，严格按照此结构输出，占位符 `[...]` 替换为实际内容，无法确认的信息填写"待确认"。

标注 **[核心]** 的章节必须输出；标注 **[可选]** 的章节按需输出。

````markdown
# 交互规格：[页面/功能名称]

## 1. 页面目标 [核心]

- 页面/菜单名称：
- 目标用户：
- 用户打开此页面后可以完成的核心任务：
- 推荐 IIDP 实现方式：标准模板/在线视图/hook/扩展视图/Vue2 组件
- 是否需要前端代码：是/否

## 2. 用户流程 [核心]

```text
用户进入页面
  → 系统加载菜单、权限和视图
  → 展示 [初始状态]
  → 用户操作 A：[查询/新增/编辑/删除/导入/导出/选择]
    → 成功：[刷新列表/关闭弹窗/展示提示/跳转]
    → 失败：[字段校验/权限提示/平台错误/停留当前状态]
  → 用户操作 B：
    → ...
```

### 2.1 关键路径说明

| --- | --- | --- | --- | --- |
| [路径名] | [触发条件] | [成功结果] | [失败处理] | [业务规则/校验] |

填写示例：

| 删除记录 | 点击删除按钮 → 确认 | 刷新列表、提示成功 | 显示平台错误 | 有 delete 权限、状态允许删除 |

### 3.1 页面级状态
````

## 审核重点

交互规格审核时检查：

- [ ] 是否只描述交互行为，没有输出节点 id、selector、ds_config、bind_ 等技术细节
- [ ] 用户流程是否覆盖主要成功路径和关键失败路径
- [ ] 状态表是否包含权限态、错误细分（字段校验/业务规则/服务异常）、保存中态
- [ ] 多模型场景（如有）是否说明了主子表联动方式
- [ ] 响应式策略是否明确到具体区域
- [ ] 可访问性检查项是否可操作、可验证
