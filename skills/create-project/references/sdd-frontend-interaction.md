# 前端交互设计规格

前端交互设计规格专注于行为、状态、数据流和验收，不追求像素级还原。当前 IIDP 项目中，旧版通用前端 Spec 里的组件层级、Props、事件和状态必须转换为 IIDP 节点树、数据源、绑定、事件、hook、commands 和后端契约。

## 交互规格核心要素

| 要素 | IIDP 写法 | 示例 |
|---|---|---|
| 用户流程 | 进入页面、加载视图、查询、新增、编辑、删除、导入导出、异常处理 | 进入合同页面 → 查询合同 → 打开新增表单 → 保存后刷新列表 |
| 组件层级 | 页面节点树 | `page > container > search/grid/form/dialog/button` |
| Props 接口 | 节点属性、`dataSource`、`ds_config.options`、后端视图字段 | `bind_value`、`bind_tableData`、`options.params` |
| 交互状态 | 状态清单和承载方式 | default/loading/empty/error/disabled/success |
| 响应式行为 | 标准模板容器、`container`、`row`、`span`、弹窗滚动 | 表单 24 栅格、表格固定高度、弹窗内部滚动 |
| 可访问性 | 表单 label、错误文字、键盘操作、危险操作确认 | 必填提示、删除确认、权限不可用说明 |
| 验收标准 | 可操作、可验证、可追踪 | 节点 id 已确认、服务参数一致、失败态可触发 |

## 页面级交互规格模板

````markdown
# 交互规格：[页面/组件名称]

## 1. 页面目标

- 页面/菜单名称：
- 目标用户：
- 用户打开此页面后可以完成的核心任务：
- 推荐 IIDP 实现方式：标准模板/在线视图/hook/扩展视图/Vue2 组件

## 2. 用户流程

```text
用户进入页面
  → 系统加载菜单、权限和视图
  → 展示 [初始状态]
  → 用户操作 A：[查询/新增/编辑/删除/导入/导出/选择]
    → 成功：[刷新列表/关闭弹窗/展示提示/跳转]
    → 失败：[字段校验/权限提示/平台错误/停留当前状态]
  → 用户操作 B：
    → ...
```

## 3. IIDP 节点层级

```text
page: [菜单或页面 key]
└── container: [页面主容器 id]
    ├── search: [搜索区视图或节点 id]
    ├── grid/table: [表格视图或节点 id]
    │   ├── tbar/button: [工具栏按钮]
    │   └── operation/button: [行操作按钮]
    ├── form/dialog/drawer: [表单或弹窗节点]
    └── extension/container: [扩展节点]
````

## 4. 交互状态表

| 节点/区域 | 状态 | 触发条件 | 展示效果 | 数据/服务 | IIDP 承载方式 | 验收 |
|---|---|---|---|---|---|---|
| [节点] | default | 页面加载完成 | [效果] | loadView/search | 标准模板 | [检查点] |
| [节点] | loading | 请求中 | loading/按钮禁用 | search/save | 标准模板/hook | [检查点] |
| [节点] | empty | 查询结果为空 | 空态文案 | search | 标准模板/扩展视图 | [检查点] |
| [节点] | error | 请求失败 | 错误提示 | service | 后端异常/hook | [检查点] |
| [节点] | disabled | 权限或状态不允许 | 置灰/隐藏/提示 | auth/status | 权限配置/服务校验 | [检查点] |

## 5. 节点属性与数据契约

| 节点 | type | id | selector | 数据源 | 绑定 | 事件/命令 | 后端契约 |
|---|---|---|---|---|---|---|---|
| [节点名] | table/form/button | [id/待确认] | [selector] | `ds_config.name` | `bind_`/`bind_two_` | `bind_on_`/`commands` | model/service/args |

规则：
- `id` 必须有来源：用户提供、标准模板规则库、代码/页面验证；不能按文案猜。
- `selector` 必须说明命中目标和验证方式。
- `ds_config` 必须说明 `type/name/autoRequest/options/reqPrep/reqAfter`。
- `bind_on_` 事件名必须是真实组件事件，不臆造事件名。
- hook 必须说明路径、是否调用 `vm.super`、读写哪些 `vm.biz` 字段。

## 6. 响应式、容器和滚动

| 场景 | 规则 |
|---|---|
| 标准后台页面 | 优先继承标准模板容器，不做大面积自定义布局 |
| 表单布局 | 使用 `row`、`span`、`form-container`，避免固定宽度破坏自适应 |
| 表格区域 | 明确高度、分页、滚动归属和空态 |
| 弹窗/抽屉 | 明确尺寸、内部滚动、关闭方式、保存成功刷新策略 |
| iframe/TABIFRAME | 只有平台能力或性能场景需要时才启用，并标记依据 |

## 7. 可访问性与可用性

- 表单字段有清晰 label、必填标识和错误文字。
- 错误提示不能只依赖颜色。
- 删除、批量操作、状态变更必须有确认或后端二次校验。
- 权限不足时说明隐藏、置灰或服务端拒绝的策略。
- 加载、空态、异常态必须可触发和可验证。

## 8. 验收标准

- [ ] 用户流程中的所有成功和失败路径都有状态说明。
- [ ] 页面节点树完整，目标节点 id 和 selector 来源明确。
- [ ] 数据源、绑定、事件、commands 与后端契约一致。
- [ ] 标准模板/在线视图能覆盖的部分没有被误写成自定义页面。
- [ ] hook、扩展视图或 Vue2 组件的使用理由清晰。
- [ ] 响应式、容器、滚动、弹窗/抽屉行为已说明。
- [ ] 权限、异常、空态、加载态有可验证结果。

## 9. 待确认事项

- [原型来源/MCP 资源 URI/节点 id/selector/事件名/服务/权限码/状态枚举]
```

## 实现分支建议

```text
传统后台 CRUD、搜索、表格、表单
  → 标准模板/在线视图

查询、保存、删除、权限、错误提示等标准流程增强
  → hook

插入按钮、调整节点、补充弹窗、合并属性
  → 扩展视图

IIDP 节点难以表达的复杂内部状态
  → 自定义 Vue2 组件
```

## 审核重点

- 是否只描述视觉，没有描述用户流程和状态。
- 是否把旧版 Props 原样写成 TypeScript，而没有转换为 IIDP 节点属性和数据契约。
- 是否缺少节点 id 来源、selector 验证方式、`ds_config` 参数和后端服务。
- 是否把标准模板可以完成的后台页面误判为自定义 Vue2 组件。
- 是否遗漏 loading、empty、error、disabled、permission denied 等状态。
