# 扩展开发规范
## 1 关于主题扩展

  对页面顶部，侧边栏，菜单等公共模块的扩展

  需独立新增app内扩展，可以单独自行安装卸载，以免影响全局

## 2 关于自定义组件


  自定义组件名称使用应用名称+功能名称

  例：tech-iot-echarts, tech-rgdd-map

## 3 关于扩展名称

	扩展名称使用应用名称+菜单名+功能名称

	例：iot_iiot_program_menu_search_extend
	
## 4 关于样式的扩展

	对框架使用的Element的样式修改时，使用局部作用域，以免影响全局样式

## 5 关于全局变量，全局方法

	全局变量，全局方法名称使用应用名称+功能名称
  ```js
	例: 
    const IOT_THEME_OPTIONS = ['blue', 'red', 'green']

    function iotGetSearchParams(params){
      console.log(params)
    }
  ```
## 6 关于扩展选择的节点id

	在选择扩展的节点id时，对于id中包含undefined的节点，可能是平台的异常，请咨询平台人员	

## 7 关于扩展注释

  在对某节点扩展时，补充注释信息，说明页面节点，功能描述等信息，方便阅读、定位
