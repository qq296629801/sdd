# 扩展类型

> 所有 type 都会判断有定义 beforeOperate 前置钩子函数就会去执行，前置钩子函数顾名思义是在执行对应 type 的处理操作前执行。

扩展的 id 也支持 id 选择器写法，详细用法：[id 选择器写法](/pages/8e3533/)

> beforeOperate 参数说明

```js
demo_example_item_menu_tab_itemAttributeList_table_main_table: {
    selector: {
        attr: 'id',
        // 此处id也支持id选择器写法 例如'menu_container_table_search items_1'
        value: 'demo_example_item_menu_tab_itemAttributeList_table_main_table'
    },
    type: 'replace',
        // useOriginElement: false, // 是否使用 beforeOperate方法里面 options参数的originElement （可选参数） 默认true  因为参数是克隆出来的 因为克隆对页面逻辑有影响可以配置 false
        /**
         * 函数描述：处理视图扩展前置钩子函数
         *
         * @param app 总应用参数,
         *          可以通过 app.page.getNode('节点id') 获取之前已经渲染好的节点
         *          可以通过 app.updatePageNode(新增节点视图对象, 新增节点对象父级节点对象, (vm)=>{ 处理新视图拼接逻辑 })
         * @param operate 当前extendView视图扩展配置项
         *          operate.view 为当前外层配置的view视图对象
         * @param options 更多选项
         *          options.element 为当前selector选中的视图对象
         *          options.originElement 为当前selector选中的视图对象的初始状态对象 (非初始状态对象可能运行里面created方法动态改变了结构)
         *          options.parent 为当前selector选中的视图对象的父级对象
         *          options.mountedNode 当前扩展展开的挂载点 可以当上下文使用
         * @return view 返回新处理的视图 会替换外部配置属性上的view
         */
        beforeOperate: (app, operate, options) => {
        // 当前例子是 把当前选中节点的初始状态放进新view的items孩子中
        // return的新view replace替换到原来选中节点中
        options.originElement.name = '基础信息';
        operate.view.items.unshift(options.originElement);
        return operate.view;
    },
    view: {
        type: 'tabs',
        id: 'tabs_001',
        name: '标签页',
        activeId: '',
        items: [
            {
                id:'test_001',
                name:'标签2', // 选项卡标题
                type: 'button',
                value: '测试'
            }
        ]
    }
}
```

### type 类型

| 类型    | 说明                                                                                        |
| ------- | ------------------------------------------------------------------------------------------- |
| before  | 在选中节点前面插入视图                                                                      |
| after   | 在选中节点后面插入视图                                                                      |
| merge   | 选中节点与新 view 视图进行属性合并 若选中节点有 items 则不合并 items 属性其他属性都深度合并 |
| unshift | 在选中节点的 items 头部插入视图                                                             |
| replace | 替换选中节点为视图节点                                                                      |
| append  | 在选中节点的 items 尾部插入视图                                                             |
| delete  | 删除选中节点                                                                                |
| custom  | 对选中节点不做任何处理但会执行 beforeOperate 函数，用户自行动态构造                         |
