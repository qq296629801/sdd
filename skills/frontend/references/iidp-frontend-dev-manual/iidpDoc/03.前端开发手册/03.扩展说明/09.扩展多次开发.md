## 概述

扩展多次开发用于满足以下场景：丙方为乙方开发了工程应用 app，之后乙方再次为甲方对同一个扩展点 app 进行二次扩展开发。

通过 `arrange` 编排协议，可支持二次开发、三次开发等多层级扩展需求。

## arrange 编排协议

### 运行顺序

扩展名按字母或数字**从小到大**的顺序依次运行。例如 `xxx_extend_view_dev2` 会在 `xxx_extend_view_dev` 之后运行。

> 可在浏览器控制台执行以下代码验证排序：
> ```js
> ['xxx_extend_view_dev2', 'xxx_extend_view_dev'].sort()
> ```

### 协议结构

```js
xxx_extend_view_dev2: {
  selector: ...,
  type: ...,
  arrange: {
    // key：扩展 view 中的字段路径
    // value：编排协议类型
    'css': 'concat',        // 字符串类型：将上一次扩展与当前扩展的字符串拼接
    'ttt.testFn': 'concat', // 方法类型：先执行上一次扩展的方法，再执行当前扩展的方法

    // value 也可定义为自定义函数，完全控制合并逻辑：
    // 参数 ops: { lastView（上一个扩展处理的 view）, currentView（当前扩展处理的 view）, path }
    // 'ttt.testFn': (app, ops) => {
    //   const lastParam = window.Tech._get(ops.lastView, ops.path);
    //   const curParam = window.Tech._get(ops.currentView, ops.path);
    //   window.Tech._set(ops.currentView, ops.path, async (...args) => {
    //     await lastParam(...args);
    //     return await curParam(...args);
    //   });
    // }
  },
  view: {
    css: '.test1{color: red;}',
    ttt: {
      testFn
    }
  }
}
```

## 示例

### 一次开发（test_arrange_extend_view）

在节点 `rbac_role_menu_form_main_detail_top_common_items_0_items_name` 上，通过 `merge` 类型扩展修改了 `css` 和 `ttt.testFn`：

```js
test_arrange_extend_view: {
  selector: {
    attr: 'id',
    value: 'rbac_role_menu_form_main_detail_top_common_items_0_items_name'
  },
  type: 'merge',
  beforeOperate: (app, operateItem, options) => {
    return operateItem.view;
  },
  view: {
    css: '.test1_class{color:red;}',
    ttt: {
      testFn: async (a, b) => {
        console.log(' == in test1 == ');
        return b;
      }
    }
  }
}
```

### 二次开发（test_arrange_extend_view_demo）

在同一节点上再次扩展，并配置 `arrange` 协议为 `concat`，使两次扩展的结果进行合并：

```js
test_arrange_extend_view_demo: {
  selector: {
    attr: 'id',
    value: 'rbac_role_menu_form_main_detail_top_common_items_0_items_name'
  },
  type: 'merge',
  beforeOperate: (app, operateItem, options) => {
    return operateItem.view;
  },
  arrange: {
    css: 'concat',
    'ttt.testFn': 'concat'
    // 自定义合并函数写法（等效于 'concat'）：
    // ops: { lastView, currentView, path }
    // 'ttt.testFn': (app, ops) => {
    //   const lastParam = window.Tech._get(ops.lastView, ops.path);
    //   const curParam = window.Tech._get(ops.currentView, ops.path);
    //   window.Tech._set(ops.currentView, ops.path, async (...args) => {
    //     await lastParam(...args);
    //     return await curParam(...args);
    //   });
    // }
  },
  view: {
    css: '.test1_class{color:red;}',
    ttt: {
      testFn: async (a, b) => {
        console.log(' == in test2 == ');
        return b;
      }
    }
  }
}
```

### 运行结果

最终执行顺序为：先运行 `test_arrange_extend_view`（一次扩展），再运行 `test_arrange_extend_view_demo`（二次扩展）。

由于配置了 `arrange: concat`，合并效果如下：

- **css**：两段样式字符串拼接，结果为 `'.test1_class{color:red;}.test1_class{color:red;}'`
- **ttt.testFn**：自动包装为依次调用，等效于：
  ```js
  async (...args) => {
    await 上一次的testFn(...args);
    return await 当前的testFn(...args);
  }
  ```
