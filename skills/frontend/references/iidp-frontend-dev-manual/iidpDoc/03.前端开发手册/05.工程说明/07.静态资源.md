# 静态资源

前端静态资源(不跟随工程打包改变的大图片 或 js等大文件资源)
放 static-resource 目录(固定目录，不可更改)

资源地址： /static-resource/`自己的扩展应用的应用名`/`static-resource文件夹下的资源路径`
```js
// 视图配置例如
{
  type: 'container',
  style: {
    width: 200,
    height: 200,
    'background-image': 'url(/static-resource/base/login.png)'//base是当前app的名字
  }
}
```
页面静态资源加载优化--不需要的资源不加载
## 静态资源懒加载
1. 不配置懒加载，则所有页面默认加载该app
2. 配置后只在对应菜单激活时加载改app
3. 适配所有场景，不只在tab页iframe场景下有效

/app/quality(应用)/config/app.json
```json
{
  "global": {},
  "effectPaths": {
    // 只在符合该条件时加载当前app的静态资源
    "includeRegExp": "^/quality/"
  }
}
```
