# 数据源

1. 提供一个快捷的获取数据的方式，将数据缓存在视图上
2. dataSource 对应的是 vm.$ds

```js
{
	type: 'xxx',
	dataSource: {} // 数据源的实际作用属性，当动态数据获取到值时会注入dataSource中
	ds_config: {} // 数据源配置，可对接技术平台后台（元模型）接口和自定义api接口
}
```

## 元模型数据源

1. 配置数据源-基本

```js
{
	ds_config: {
		type: 'meta',  // 调用元模型引擎API获取数据
		name: 'data1', // 数据源标识，获取数据后将根据此名称存入dataSource.data1中
		autoRequest: true, // 在节点渲染后，是否自动触发请求，默认false
		options: {
			params: { // 参考元模型api传参规范
				args: {
					properties: ['name']
				},
				service: 'search',
				model: 'xxx'
			}
		}
	}
}
// 当以上数据源获取或者更新数据的时候会等价于
{
	dataSource: {
    data1: {/** 接口返回的数据 */}
  }
}
```

2. 手动触发数据源，设置`autoRequest: false`，关闭自动触发

```js
// 根据name触发获取数据
// params选填，不需要修改api传参时可以直接调用
// 可选入参： 若params包含app参数 例如 params.app = 'base' 则可以固定分布式app参数
// 若没有配置则从loadView或loadView接口承继下来的app
vm.request('data1', params?)

// 传参是针对配置项options.params 的深度合并
vm.request('data1', {
	args: {
		filter: [["id", "=", "xxxx"]]
	},
	model: 'aaa'
})

// 合并后params结果如下
{
	args: {
		filter: [["id", "=", "xxxx"]], // 被合入的内容
		properties: ['name']
	},
	service: 'search',
	model: 'aaa' // 被合入的内容
}
```

3. 数据源缓存
   以上数据源获取数据后将根据配置项的 name 标识缓存到 dataSource 中

```js
{
	dataSource: {
		data1: {
			a: 1,
			b: 2
		}
	}
}
```

5. 使用数据

```js
{
	type: 'text',
	bind_value: '$ds.data1.a'
},
{
	type: 'text',
	bind_value: '字符运算示例：字符串${$ds.data1.a}'   //${}支持四则运算，逻辑运算，函数等语法，详见后文
}
// js方式获取数据
vm.$ds.data1
```

6. 数据源的 dataSource 属性是可继承的

```js
{
	type: 'container',
	dataSource: {
		a: {a:1},
		b: {b:1}
	},
	items: [
		{
			type: 'container',
			dataSource: {
				a: {a:2}, // 覆盖父级数据源a
				c: {c:2}
				// 此处可以获取到父级数据源b
			},
			items: [
				type: 'container',
				created: () => {
				console.log(vm.$ds.a, vm.$ds.b, vm.$ds.c)
					// {a:2}, {b:1}, {c:2}
				}
			]
		}
	]
}
```


7. 数据源的 dataSource 断开继承配置
> 数据源 dataSource 与 $ds 调用是继承父级数据，如果想断开与父级数据的继承关系则需要配置 inheritData:false

```js
{
	type: 'container',
	dataSource: {
		a: {a:1},
		b: {b:1}
	},
	items: [
		{
			type: 'container',
      inheritData: false, // 配置断开数据源继承
			dataSource: {
				a: {a:2},
				c: {c:2}
			},
			items: [
				type: 'container',
				created: () => {
				console.log(vm.$ds.a, vm.$ds.b, vm.$ds.c)
						// {a:2}, undefined, {c:2}
				}
			]
		}
	]
}
```

> 元模型数据源发起 http 协议的请求头为：

```
请求方法: POST
Content-Type: application/json
```

> 元模型数据源发起 http 协议的响应头为：

```
Content-Type: application/json
```

## 第三方 API

> 只有个别配置和元模型数据源不一致，其余配置和用法一致

```js
{
	ds_config: {
		type: 'api',  // 调用第三方API获取数据
		name: 'data2'
		options: {
			axios: { // 支持axios(config)约定的所有内容
				method: 'post',
				url: '/user/12345',
				data: { firstName: 'Fred', lastName: 'Flintstone' },
				headers: {'X-Requested-With': 'XMLHttpRequest'}
			}
		}
		// 其余配置同元模型数据源
	}
}
```

## 静态数据源

> 直接配置就可以用，使用方法和元模型数据源一致

```js
{
	dataSource: {
		data3: {a:1, b: 1}
	}
}
```

## 完整配置

```js
// 配置数据源--完整
{
	ds_config: {
		type: 'meta',  // 调用元模型引擎API获取数据
		name: 'data1', // 数据源标识，获取数据后将根据此名称存入dataSource.data1中
		options: {
			params: { // 参考元模型api传参规范
				args: {},
				service: '',
				model: ''
			}
		},
		// 以下属性非必须 第三方API也通用
		autoRequest: false, // 在节点渲染后，是否自动触发请求，默认false
		reqPrep: (vm, options, config) => {}, // 调用接口前执行，
		reqAfter: (vm, res, config) => {return res}, // 调用接口后执行，可修改数据格式，必须返回res
		isSuccess: (vm, res, config) => {}, // 判断是否成功
		success: <(vm, res, config) => {}> | <{showTime: 3000, msg: '操作成功'}>, // 支持function和JSON，返回指定格式，替换默认提示
		error: <(vm, res, config) => {}> | <{showTime: 3000, msg: '操作失败'}>,  // 支持function和JSON，返回指定格式，替换默认提示
	}
}
```

## 数据保存

其实本身就是数据源，只是多了一个配置，调用结果也不会在 dataSource 里面显示

```js
{
	ds_config: {
		type: 'meta',
		name: 'data3',
		method: 'save' // 标识改配置是保存逻辑，调用后的结果不会缓存到dataSource中
		options: {
			params: {
				args: {
					properties: ['name']
				},
				service: 'search',
				model: 'xxx'
			}
		}
	}
}
```
