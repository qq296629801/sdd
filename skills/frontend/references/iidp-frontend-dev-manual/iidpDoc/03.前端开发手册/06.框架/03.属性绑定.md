# 属性绑定
一个节点的属性变化可以触发另一个节点的属性变化
## 示例
input输入值 => 改变form.name => 触发下个节点的改变内容<br/>
input输入1时 => 触发下个节点的隐藏
```js
{
  type: 'form',
  dataSource: {
    form: { // form表单数据，对应 $ds.form
      name: ''
    }
  },
  formConfig: {
    labelPosition: 'right'
  },
  items: [
    {
      type: 'row',
      items: [
        {
          type: 'input',
          text: '姓名',
          placeholder: '数据同步，且输入 1 时隐藏第二项',
          name: 'name' // 在form表单内会默认双向绑定表单数据的name属性，form.name
        },
        {
          type: 'text',
          bind_display: '${$ds.form.name != 1}', // 当啊name值等于1时，隐藏当前节点
          value: '同步姓名',
          bind_value: '$ds.form.name' // 绑定表单数据的name，name值改变后，value值也改变
        }
      ]
    }
  ]
}
```

## 基础语法
1. 单向绑定
	`bind_属性名`
	如`bind_value`，初始化后会转换为节点的`value`属性
```js
{
	bind_value: '$ds.data1.a'
}
// 渲染时等价于
{
	value: null
}
当
// 当数据源的data1.a值发生改变时将赋值给下面的value

```
2. 双向绑定
  * `bind_two_属性名`
  * 当数据源的data1.a值发生改变时将赋值给下面的value
  * 当改节点的value属性发生主动改变时，会同时触发$ds.data1.a值改变
```js
{
	bind_two_value: '$ds.data1.a'
}
// 当数据源的data1.a值发生改变时将赋值给下面的value
// 当改节点的value属性发生主动改变时，会同时触发$ds.data1.a值改变
```
## 属性绑定-简写
1. 数据源的dataSource也是属于属性，可被绑定，请使用简写$ds，同时也可配合\${}语法
2. 自身节点的属性也可以被绑定，请使用简写$self，同时也可配合\${}语法
3. \${}语法内还可以执行\$cmd，\$cmd代表组件的commands属性，详见后文
#### 单向绑定
```js
// 运算示例
{
	type: 'text',
	bind_value: '字符运算示例：字符串${$ds.data.a}； 运算结果${$ds.data.b + $ds.data.c}'
},
// 函数示例，必须return
{
	type: 'text',
	bind_value: '字符函数示例：${let t = 5; return t + $ds.data.b;}'
},
// 可用范围示例
{
	type: 'text',
	bind_value: '字符取值示例：${$ds.data.a}，${$self.type}, ${$cmd.fn()}'
},
// 按原变量类型返回
{
	type: 'text',
	style: {display: 'block'},
	bind_value: '${$ds.data.b}',
	bind_display: '${$ds.data.b === 1}'
},
// 省略花括号
{
	type: 'text',
	style: {display: 'block'},
	bind_value: '$ds.data.b'  // 绑定数据源
},
{
	type: 'text',
	style: {display: 'block'},
	bind_value: '$self.type'  // 绑定自己节点的属性
}
```
#### 双向绑定
> text属性值改变时会触发$ds.data.b改变
```js
{
	bind_two_value: '$ds.data.b'  // 绑定数据源
}
```
## 属性绑定完整写法

> 完整写法本质也是“监听 + 同步”的机制：当 selector 指向的节点（或其 dataSource）在 path 对应字段发生变化时，会触发本节点对应属性的重新计算与更新（等价于监听该字段变化后更新属性）。
>
> 完整写法如果 selector 不是自身节点而是其他节点，则 path 不能用 $ds，要用 dataSource

```js
{
  type: 'form',
  dataSource: {
    form: { // form表单数据，对应 $ds.form
      name: ''
    }
  },
  formConfig: {
    labelPosition: 'right'
  },
  items: [
    {
      type: 'row',
      items: [
        {
          type: 'input',
          text: '姓名',
          placeholder: '数据同步，且输入 1 时隐藏第二项',
          name: 'name' // 在form表单内会默认双向绑定表单数据的name属性，form.name
        },
        {
          type: 'text',
          bind_display: '${$ds.form.name != 1}', // 当啊name值等于1时，隐藏当前节点
          value: '同步姓名',
          bind_value: { // 绑定表单数据的name，name值改变后，value值也改变
            path: '$ds.form.name',
            selector: '', //选择目标节点，绑定目标节点的dataSource，语法参考selector
            transform: (params) => {
              // params.self 当前节点的实例
              // params.related 关联节点的实例，在此表示数据源form所在的节点实例
              return params.value + '自定义值'  // 返回值将作为value属性的值
            }
          }
        }
      ]
    }
  ]
}

```
监听$ds中的数据变化并同步的最佳实践

```js
{
  bind_two_value: { // 绑定表单数据的name，name值改变后，value值也改变
    path: '$ds.form.name',
    selector: 'xxx_other_node'
    transform: (params) => {
      // params.self 当前节点的实例
      // params.related 关联节点的实例，在此表示数据源form所在的节点实例
      return params.value + '自定义值'  // 返回值将作为value属性的值
    }
  }
}
```

绑定指定节点的数据
```js
{
	bind_value: { // 绑定表单数据的name，name值改变后，value值也改变
		path: 'dataSource.form.name', // 不是自身节点 是其他节点，则path不能用$ds要用dataSource
		selector: 'xxx_other_node', //选择目标节点，绑定目标节点的dataSource，语法参考selector
		transform: (params) => {
			// params.self 当前节点的实例
			// params.related 关联节点的实例，在此表示数据源form所在的节点实例
			return params.value + '自定义值'  // 返回值将作为value属性的值
		}
	}
}

```
