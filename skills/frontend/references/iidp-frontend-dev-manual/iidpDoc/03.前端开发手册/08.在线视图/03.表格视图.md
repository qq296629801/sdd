# 表格视图

- columns：表格列。
- buttons：表格操作列按钮。
- tbar： 工具条按钮。

更多配置请看[后端-基本视图-grid](/pages/405eb0/#_3-8-1-2-表格-grid)

```js
{
  "type": "grid",
  "oSize": 200, // 每次渲染的数据偏移量，偏移量越大白屏几率越小，渲染次数就越少，但每次渲染耗时就越久。 默认2
  "transformTableData": "(list)=>{list.forEach((item)=>{item.xxx=item.xxx/1000});return list}",  //配置初始显示数据转换
  "isDefaultSelectRow": true,   //单主表模式是否默认选中第一行
  "advancedQueryGroupSize": 2,  // 限制条件组的数量，小于0或不配置则就不限制
  "advancedQueryConditionSize": 2, // 限制条件组的条件数量，小于0或不配置则就不限制
  "columns": ["name", "login", "mobile", "email", "status"],
  "buttons": [
    {
      "name": "编辑",
      "action": "edit",
      "auth": "update"
    },
    {
      "name": "重置密码",
      "auth": "resetPassword",
      "service": "resetPassword",
      "action": "openView",
      "model": "rbac_user",
      "views": "custom",
      "args":{},
    }
  ],
  "tbar": [
    {
      "name": "新增",
      "action": "create",
      "auth": "create"
    },
    {
      "name": "删除",
      "action": "delete",
      "auth": "delete"
    }
  ]
}
```

## 高级查询

可以通过`apps.json`进行配置开启全局高级查询

```js
{
    "techAdvancedTable": true,  // 需要开启高级表格
    "techAdvancedQuery": true,  // 开启高级查询
```

## 表格行 key 配置 (可选配置)

- 使用行 key 能提供渲染性能 可选配置 在没有配置情况下 默认 "useKey": true "keyField": "id"

```js
{
  "type": "grid",
  "gridOptions": {
    "row-config": {
      "useKey": true, // 是否使用行key 默认true  若表格上需求是存在两行数据一样(包括id) 则配置false
      "keyField": "id" // 取当前行的某属性作为行key 默认为id属性
    }
  }
}
```

```js

  "editSyncCheckBg": true, //可编辑表格配置选中添加背景色
  "gridEditMode":{
    "mode": "multipleRow", // multipleRow:多行编辑 singleRow:单行编辑

    "addRowAfter": true, // 行内编辑新增行时添加到表格末尾,默认为false，添加到表格头部
    "arrowKeys": true, // 是否启用上下左右键控制正在编辑的单元格 默认开启
    "btnConfig": {// 仅用于配置在子表的tbar编辑按钮，由主表选择的数据控制该按钮的禁用状态状态
```

## 自定义操作列按钮

- enableCondition: 根据函数返回值(true 或 false)设置按钮禁用状态

- showCondition: 根据函数返回值(true 或 false)动态显示或隐藏按钮

- actionAfter: "refreshTable" 接口执行后刷新操作，接口必须返回成功标识（返回 true 即可）

- actionBeforeFn: 接口执行前的自定义操作，返回 true 继续执行接口调用，返回 false 终止执行接口调用

- actionAfterFn: 接口执行后的自定义操作

- isLoading: true 按钮点击后，按钮自身添加 loading

- showPopLoading: true 按钮点击后，页面全局添加 loading

- args 说明:

  $row.name: 当前表格行数据

  $table.row: 主表格当前行数据，用于主列表 tbar 配置

  $table.subRow: 子表格当前行数据，用于上下表子表 tbar 配置

  $form.curForm: 主表单当前数据，用于主表单子表 tbar 配置

  自定义参数 eg: "xxx": "123"

  自定义传参 函数 othersParams: (vm) => { return {xxx:123}} 动态构造 args 方法

```js
{
  "type": "grid",
  "columns": ["name", "code", "is_admin" ],
  "buttons": [
    {
      "name": "发布",
      "service": "installApp",
      "auth": "update",
      "model": "meta_app_store",
      "app": "sie-iidp-iam", // 可不配置，默认当前菜单所属app
      "customStyleFn": "(row)=>{ if(row.name==='123'){ return { fontSize: '12px', color: '#333' } } }", //根据当前行数据自定义按钮的样式
      // 按钮自定义传参
      "args":{
        "name1": "$row.name", // row 为当前行数据
        "name2": "$table.row.name",
        "name3": "$table.subRow.name",
        "name4": "$form.curForm.name",
        "xxx": "123",
        "othersParams": "(vm) => { return {xxx:123}}"
      },
      "enableCondition": "(row,vm)=> {if (row && row[0].rowIndex == 0) {return false} else { return true}}",
      "showCondition": "(row,button,vm)=> {if (row.rowIndex == 0) {return false} else { return true}}",
      "actionBeforeFn": "(vm) => {console.log(vm); return false}",
      // 配置了actionAfter=refreshTable 点击按钮后会自动刷新表格
      "actionAfter": "refreshTable",
      "actionAfterFn": "(vm, data) => {console.log(vm)}" // data为接口返回的数据
    }
  ],
  "tbar": [
    "@defaults"
  ]
}


```

```js

{
  "type": "grid",
```

```js
      "checkboxTreeNode": true,  //在复选框列展开子节点配置
      "lazy": true,    //配置是否懒加载
      "hasChild": "hasChildren",

        const table = tech_app.page.getNode('mes_workorder_t_menu_table_main_table');  //获取表格节点，调用定义好的接口
        const res = await table.request('tableChildData', {

        return res.data.length>0?res.data:[];
```

## 表格 tbar 自定义按钮

- bind_disabled: 根据选择行数据控制按钮禁用状态

  - $ds.selectedTableData: 当前表格选择的行数据
  - $ds.checkedDataList: 当前表格勾选的行数据（配置了 checkbox: multiple_2）
  - $ds.checkedDataIds: 当前表格勾选数据的 id 集合

- bind_display: 动态控制工具栏按钮显示或隐藏（表达式返回 true 显示，false 隐藏）

- args 说明:

  - "bind_ids": "$ds.checkedDataIds": 当前表格勾选数据的 id 集合
  - $table.row: 主表格当前行数据，用于主列表 tbar 配置
  - $table.subRow: 子表格当前行数据，用于上下表子表 tbar 配置
  - $form.curForm: 主表单当前数据,用于主表单子表 tbar 配置
  - 自定义参数: `"xxx": "123"`
  - 自定义函数传参,动态构造 args 方法: `othersParams: (vm) => { return {xxx:123}}`

- actionBeforeFn: 接口执行前的自定义操作，返回 true 继续执行接口调用，返回 false 终止执行接口调用

- actionAfter: "refreshTable"
  接口执行后刷新操作，接口必须返回成功标识（返回 true 即可）

- actionAfterFn:接口执行后的自定义操作

```js
{
    "action": "xxx",
    "service": "xxx",
    "auth": "xxx",
    "name": "自定义操作",
    "model": "xxxxxx",
    "bind_disabled": "${$ds.selectedTableData.length === 0 || $ds.selectedTableData[0].state.value !== '0'}",
    "args": {
        "bind_ids": "$ds.checkedDataIds",
        "name": "$table.row.name",
        "name1": "$table.subRow.name",
        "name2": "$form.curForm.name",
        "xxx": "123",
        "othersParams": "(vm) => { return {xxx:123}}"
    },
    "actionBeforeFn": "(vm) => {console.log(vm); return false}",
    "actionAfter": "refreshTable",
    "actionAfterFn": "(vm) => {console.log(vm)}"
}


```

## 使用表格原生方法

表格是在 vxe-grid 组件的基础上封装的，因此可以调用组件自带的方法。
调用步骤：

1. 到 vxe 官网根据自己的需求找到对应的方法 所有方法都在 Methods 里
2. 找到表格实例，例如 vm.instance
3. 调用方法：vm.instance.getInstance().xxx(); xxx 是官方提供的方法

   [vxe 官网地址](https://vxetable.cn/v3/#/grid/api)

4. 在使用 vxetable3 原生配置
   > https://vxetable.cn/v3.8/#/table/api
   > 例如：在 vxetable3 官网 api 上搜索
   > ![vxetable3官网api上搜索](./img/vxetableSearch.png)
   > 例如：搜索到配置为： (按需禁用行勾选)
   ```js
    "checkbox-config": {
      checkMethod: ({ row }) => { return false }
    }
   ```
   则在表格视图配置为：
   ```js
   {
    。。。
    type: 'grid',
    gridOptions: {
      "checkbox-config": {
        checkMethod: ({ row }) => { return false }
      }
    }
    。。。
   }
   ```
